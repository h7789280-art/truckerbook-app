### Модель монетизации

**БЕСПЛАТНОГО РЕЖИМА НЕТ. Только 7-дневный trial с полным доступом, потом оплата.**
Никакого фримиума. Никаких ограничений по функциям. Никаких "3 рейса бесплатно".
Водитель получает ВСЁ на 7 дней. На 8-й день — Paywall. Точка.

| Состояние | plan | Что видит водитель |
|-----------|------|--------------------|
| Регистрация | `trial` | Полный доступ, все фичи, до 5 машин. Баннер "Осталось X дней" |
| Trial истёк | `expired` | Paywall. Приложение заблокировано. Данные сохранены. |
| Оплатил | `pro` | Полный доступ, до 5 машин |
| B2B | `business` | Полный + панель руководителя, безлимит машин |

**Логика:**
- Регистрация: `plan = 'trial'`, `trial_ends_at = now() + 7 days`
- Крон n8n (ежедневно 00:05): если `trial_ends_at < now()` и `plan = 'trial'` → `plan = 'expired'`
- Фронтенд: если `plan = 'expired'` → Paywall.jsx, ВСЕ табы заблокированы
- Оплата прошла (webhook) → `plan = 'pro'`, `paid_until = now() + 30 days`

**Paywall.jsx — что показывает:**
- "Ваш пробный период закончился"
- Статистика за trial: "За 7 дней вы записали X заправок, Y рейсов, сэкономили Z ₽ на расчётах"
- Кнопка "Продолжить за 249 ₽/мес"
- Кнопка "Годовой план — 1 990 ₽ (скидка 33%)"
- Мелким: "Ваши данные сохранены и будут доступны после оплаты"

**Баннер обратного отсчёта (во время trial):**
На главном экране: "Pro-доступ: осталось 5 дней" → жёлтый.
Последние 2 дня: "Pro-доступ: осталось 1 день!" → красный.
Нажатие → экран оплаты (можно оплатить досрочно).

**Цены:**

## Авторизация и безопасность

### Регистрация (один раз)

```
Экран 1: Телефон     → +7 XXX XXX XX XX → кнопка "Получить код"
Экран 2: SMS-код     → 4 цифры из SMS → кнопка "Войти"
Экран 3: Профиль     → Имя, Марка (выпадающий), Модель, Пробег, Госномер (необяз.), Расход (слайдер)
Экран 4: Создать PIN → Ввести 4 цифры → Повторить 4 цифры → Подтвердить
Экран 5: Добро пожаловать → Сетка фич, баннер "7 дней Pro бесплатно" → "Поехали!"
```

### Каждый вход (после регистрации)

```
Открыл приложение → PinLock.jsx → ввёл 4-значный PIN → приложение
                  → ИЛИ отпечаток/Face ID (если включил) → приложение
```

**SMS больше не нужна.** Единственное исключение — сброс PIN (кнопка "Забыл PIN" → SMS-код → новый PIN).

### Поле в profiles

| Поле | Тип | Описание |
|------|-----|----------|
| pin_hash | text | SHA-256 хэш PIN-кода. НИКОГДА не хранить PIN в открытом виде. |
| biometric_enabled | boolean DEFAULT false | Включена ли биометрия (отпечаток/Face ID) |

### Логика PinLock.jsx

1. При открытии App.jsx проверяет: есть ли Supabase сессия?
2. Нет сессии → Auth.jsx (регистрация)
3. Есть сессия → PinLock.jsx (ввод PIN)
4. PIN верный → проверка plan (trial/expired/pro) → Paywall или основное приложение
5. Biometric: если `biometric_enabled = true`, сначала предложить отпечаток, если отказ → PIN

### Блок-схема входа

```
Открыл приложение
  ├─ Нет сессии → Auth.jsx (регистрация: SMS → профиль → PIN → Welcome)
  └─ Есть сессия → PinLock.jsx
       ├─ Биометрия включена → предложить отпечаток/Face ID
       │    ├─ Успех → далее
       │    └─ Отказ → ввод PIN
       └─ Ввод PIN (4 цифры)
            ├─ Верный → проверка plan
            │    ├─ plan = 'expired' → Paywall.jsx
            │    └─ plan = 'trial'/'pro'/'business' → Основное приложение
            └─ Неверный → "Неверный PIN" (3 попытки, потом "Забыл PIN?" → SMS)
```
